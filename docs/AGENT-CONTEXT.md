# DayRecord — Agent 对话外上下文层

## 问题

Hermes agent、小龙虾、nanobot 等个人 Agent 的上下文主要来自：

- 当前对话
- 用户主动粘贴/描述
- 少量工具调用结果

它们**不知道**用户在对话之外电脑上发生了什么：正在修哪个 bug、刚读了哪篇文档、下午时间花在哪个项目、哪些决定已经做过。

## DayRecord 补什么

| 来源 | Agent 通常拿不到 | DayRecord 提供 |
|------|------------------|----------------|
| 对话内 | 用户说出口的内容 | （不替代） |
| 对话外 | 前台应用与窗口标题 | 活动时间轴 |
| 对话外 | 阅读/浏览的正文（无打字） | UIA 可见文本快照 |
| 对话外 | 键入与粘贴片段 | 输入会话 |
| 对话外 | 习惯与偏好 | 习惯画像 → `USER.md` |
| 对话外 | 项目/决策/阻塞 | 事实抽取 → `MEMORY.md` / `facts.md` |
| 对话外 | 今日工作脉络 | 日复盘 → `memories/YYYY-MM-DD.md` |

## 目标 Agent

| Agent | 接入方式（当前 / 规划） |
|-------|-------------------------|
| **Hermes Agent** | 导出 `USER.md` / `MEMORY.md` / `memories/*.md` → `~/.hermes/memories/` |
| **小龙虾** | 同上：标准 Markdown 记忆文件，拷贝或软链到 Agent 记忆目录 |
| **nanobot** | 同上；后续可提供 JSON context feed |
| **任意 Agent** | 不绑定运行时；记忆为文件，非专有 DB |

## 架构

```text
┌─────────────────────────────────────────────────────────┐
│ 采集层（本地、按需 AI）                                    │
│  键盘/粘贴 · 窗口时间轴 · UIA 文本 · 空闲/暂停控制          │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│ 存储层  dayrecord.db（SQLite）                            │
│  sessions · activities · summaries · facts · settings    │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│ 记忆层（用户触发或自动）                                   │
│  工作复盘 · 习惯画像 · 事实抽取 · 脱敏                     │
└──────────────────────────┬──────────────────────────────┘
                           ▼
┌─────────────────────────────────────────────────────────┐
│ 导出层  hermes-export/                                   │
│  USER.md · MEMORY.md · memories/*.md · facts.md          │
└──────────────────────────┬──────────────────────────────┘
                           ▼
                    Agent 启动时注入 / 检索
```

## 「GPT-3 时刻」定义

不是日报写得漂亮，而是用户只说：

> 「继续。」

Agent 能回答：

> 「你刚才在修 DayRecord 活动时间轴 checkpoint，问题是 13:34 后数据未落库；已改 `ActivityTracker`，建议重新生成复盘验证。」

**条件：** 结构化记忆 + 可检索导出 + Agent 实际读取这些文件。

## 当前 MVP 已具备

- [x] 本地活动与会话采集
- [x] DeepSeek 工作复盘（人类视图 + `memories/`）
- [x] 习惯画像 → `USER.md`
- [x] 事实抽取 → `MEMORY.md` / `facts.md`
- [x] Hermes 格式导出 + 可选自动导出

## 下一阶段（Agent Context API）

1. **`context.json`** — 机器可读当日上下文（项目、焦点、未决线程、最近决策）
2. **本地 HTTP / IPC 查询** — Agent 工具可调用：
   - `what_is_user_working_on_now`
   - `get_recent_project_state`
   - `list_open_threads`
3. **增量推送** — 导出目录文件变更时通知 Agent（watch / webhook）
4. **跨 Agent 记忆契约** — 文档化字段，小龙虾/nanobot 各写一层薄适配

## 设计原则

1. **对话外，不替代对话** — 采集默认不上云；仅用户触发的摘要/抽取走 DeepSeek。
2. **导出不含原始键入** — Agent 拿画像与事实，不拿完整键盘记录。
3. **标准 Markdown 优先** — 避免锁死在单一 Agent CLI。
4. **无截图** — UIA 文本 + 元数据；接受 Electron/Canvas 盲区，不伪造内容。
